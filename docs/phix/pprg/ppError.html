Print Preview and Report Generator

Error Messages

The following table only list a few unusual messages, not trival messages like "bad format".
There are of course several more cryptic messages not listed here which occur when an internal error occurs in the software; these should be reported as bugs.

Layout is stamped(see manual). Continue? (Warning when attempting to amend a report layout). The layout has been stamped so that it can be distributed free of charge and used free of charge. However if the report is amended, the report will need to be licenced to remove the "UNREGISTERED SOFTWARE" watermark.
Mixed paper sizes not supported. A report may not contain more than one paper size. This can only happen when several pages have been stitched together via the pageSequenceHook() routine, as shown in the booklet print example in pptest.exw. Portrait and Landscape layouts may be mixed, provided they are on the same paper size.
illegal summation on repeat field Only type 's' (section) subtotals may be repeated on a page. It makes no sense to print carried forward, brought forward, page, final, or section page numbers more than once on a page. If you want to, insert two copies of the field.
Print not allowed in Screen mode See The Zoom Window. When Screen is checked, the report details are fitted height and width to the display window. This is not a valid paper size and therefore cannot be printed.
Unexpected printer resolution error The size of A4 portrait does not appear to be 210mm by 297mm. This is either an error in the report subroutines or a printer setup error/corruption. Please report this to the author.
Printer unavailable Try rebooting the computer. Some other program may not be releasing the printer correctly. Let me know if it occurs often.
Incorrect record length for record XX The contents of datadict.txt do not agree with the datastream contents. Verify the length of each record in the datastream creation program as suggested here.
unmatched repeat depths. The report generator believes you have specified that one field should be printed say 8 times per page, and another field from the same record say 10 times. Check the Repeat values on all fields. It may also be a mistake on a calculated field, referencing the wrong record perhaps.
record xx not in alphabetic sequence at line nn Records in datadict.txt must be defined in alphabetic order. This helps prevent accidentally reusing a two-character code already in use.
more/fewer format qualifiers than fields The Format and Fields do not match. Cancel field entry/amendment and try again.
field list mishap, please check The automatic insertion of a field qualifer or a field name in the field list has noticed a problem. This can occur after manually editing the fields list, leaving the cursor in an unexpected position. Cancel field entry/amendment and try again.
missing operator?, please check The automatic insertion of a field name into the calculation expression could not find an appropriate bracket, comma, or operator (+-/* etc) at the field insertion point. It has placed a question mark where it believes the missing operator should go.
