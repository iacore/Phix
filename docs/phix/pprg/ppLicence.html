Print Preview and Report Generator

Licencing

In a nutshell, you can develop using this free, distribute it free, and end users can use unaltered reports free, but if an end user wants to tailor a report (and your application must allow them to do that), then it is not free. However it is not open source or freeware, and there are some conditions. You need to read this whole section to get the full picture.

The software is provided as a non-copy-protected and fully functional component, with an "UNREGISTERED SOFTWARE" watermark.

However, there are two methods of removing this, free of charge:

Removing the UNREGISTERED SOFTWARE watermark without registering

Whenever a report is altered, the watermark is removed for one month. There is no limit to the number of times this can be done. Hence application developers can implement and maintain reports without ever having to register, and while the report is being actively developed or regularly changed, never even see the watermark.

Application developers may also be interested in utilising the using the print preview and report generator registration mechanism to collect revenue for their own application, see Commission Codes below.

Personal users (if the report has not been stamped, see below) may choose to simply accept the watermark appearing on their personal reports, or they may make a minor amendment (eg drag a field half an inch to the right, then drag it back) to remove the watermark. The process is complete in a matter of seconds. The only catch is that you are not reminded to do this each month except when the watermark appears on the physically printed page.

Business users may also take advantage of this indefinitely; however, the cost of wasted ink, paper, and time soon outweight registration in all but the smallest commercial enterprise, especially considering that this process must normally be individually applied to each report in an application, every month.

I am assuming, of course, that during application development, and even when printing sample reports for documentation purposes, this is at most a minor inconvenience, but that is not so when trying to run a business.

Stamping

Reports may also be permanently licenced, for free, at the authors discretion.

The application must include the ability to amend the report, so that while the application can be distributed free of charge and function free of charge, if the user chooses to amend the report the licencing mechanism will be activated.

The documentation for the application should note the consequence of amending a report, but should not otherwise seek to actively discourage users from tailoring their reports (and therefore needing to register). If the user makes a backup of the layouts.edb file, experiments with layout amendment, and then restores the layouts.edb, then the report will remain stamped (but obviously any changes made will be lost). A warning message is displayed when an attempt is made to amend a stamped report.

Report Stamping is therefore a simple trade: Application Developers offer to distribute the reporting routines in exchange for not being personally charged, and not forcing any cost on users who do not need to amend the standard reports, but creating the potential for revenue.

Please check the latest version of this document to ensure this service is still available before applying.

Mail details of the report(s) requiring stamping to petelomax@blueyonder.co.uk, along with a link to an online screenshot/manual page and a download area for the layout database. Only nearly completed applications will be considered. Reports with obvious potential for ad-hoc tailoring will be given preferrential treatment.

A hash code for the layout will be added to an internal table (hard-coded within the shrouded sources) for the next release. At least a two-week turnaround should be expected.

Please note that using advanced features via the pageSequenceHook() routine disables the effect of report stamping.

Commission Codes

This section only applies to application developers. Next section

The print preview and report generator subroutines will most likely form a key element of any application which uses them. Hence if you wish to licence your application, it makes some sense to base the licencing model on the existing payment infrastructure rather than re-invent the wheel.

Software developers may pass a previously agreed three-character commission code to the setCommissionCode() routine.

This allows the registration process to alter the price to cover the cost of both their application and the reporting routines rental fee. The application developer may charge any amount for their software, the price for the print routines remains the same.

The softare is then registered in one operation, and the developer does not need to write and host the necessary cgi programs or any client side handling for registration codes, sign up with payment card processors, dynamic dns servers, etc.

Registering the Software

A software licence may be purchased for a twelve month period only.

A button on the title bar of the print preview display takes three forms:

  • A green tick on a black dollar sign indicates the software is properly licenced. If clicked, the expiry date of the licence is shown. An additional twelve-month licence may be purchased at any time to extend the current licence for a further year.

  • A green tick on a red dollar sign indicates the softare licence is due to expire.

  • A red dollar sign with no tick indicates the software is unlicenced: the watermark appears on every page of the printed reports.
Clicking on this button opens a window displaying the expiry date of the current licence (if there is one), a 'Register Now' button (unless the report is stamped), along with 'Support Query' and 'Cancel' options.

When the register button is pressed, a web browser window is opened showing typically the product code, quantity and description, and a "buy now" style button which will accept credit card details. (The exact details depend on the card payment processor, eg RegSoft, Digibuy, etc.) [In the current version this step is omitted]

When payment details have been accepted, a confirmation message is displayed in the browser as follows (eg):

Depending on the browser version, and the security settings for scripting, either:

The screen is immediately closed and the registration key applied.

The user must press the "Copy" button on the browser window.

If the button does not work, the user must manually select and copy the registration code.

Once a valid registration code is detected in the windows system clipboard, the web browser is automatically closed and the key applied without any further intervention from the user. A registration successful message is shown.

Note: If you elect to pay by phone, fax, or even by post, the report must not be altered between sending payment and entering the registration code, and must be applied to the same report within ten calendar days. In this case you may select the report which is least likely to require amendment during that time. For further details on the availability of such payment methods, follow the above procedure and the appropriate links/instructions on the payment site.
[The author makes no claim to the availability or otherwise of such alternative payment methods at this time.]

Backup

The user is strongly advised to take a backup of the layouts.edb file following successful registration.

For security reasons, the registration key can only be applied within ten calendar days. Instead of saving the registration key, the layouts.edb file should be backed up shortly after successful registration.

Integrity Checking

The software licencing mechanism also checks the layout database for corruption and unauthorised modification. If an error is detected and the override option is selected, any existing licence is void. Restore from backup to avoid this.
[The override option is to ignore the warning message and amend the layout; this clears the 'corrupt' status but also cancels any registration key.]

Hardware Upgrade

Note also that keys are tied to the particular machine and will not work when copied to another computer, or in some cases when the configuration is altered. A new 12-month registration will be required.

Use the support query function to request a refund if applicable. Do not alter the encoded details in the message body.
