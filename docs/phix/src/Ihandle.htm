Ihandle

Syntax: include pGUI.e

Ihandle identifier
Description: Type. An integer > 0.

Use this to define interface element variables. It is probably unwise to use this to determine whether an object is an Ihandle, as many things which are clearly not (eg 1, 5000, 'X') will pass.

Note that assigning NULL to a variable declared as an Ihandle will trigger a typecheck. Many of the pGUI routines yield an Ihandle or NULL; for such cases the type Ihandln (an integer >= 0) is provided. I write all new code using Ihandle exclusively, and switch individual variables to Ihandln once it becomes evidently a valid and necessary case, as opposed to getting a NULL by mistake. There are doubtless several routines in pGUI.e that specify Ihandle parameters/returns which should really be Ihandln.
Example: 
include pGUI.e

Ihandle quit_bt = IupButton("Quit", 0)
See Also: IupButton
