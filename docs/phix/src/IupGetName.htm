IupGetName

Definition: include pGUI.e

string name = IupGetName(Ihandle ih)
Description: Returns the name of an interface element, if the element has an associated name using IupSetHandle()

ih: Identifier of the interface element.

Returns: the name, or "" if no name has been associated with the element.
See Also: IupSetHandle, IupGetHandle, IupGetAllNames
Technicalia If more than one name has been associated with the specified element, it returns the first one found, in a non-deterministic fashion with no guarantee of any ordering (such as first or last, chronological or alphabetical: you may as well assume it will be random and different on each run, or at least on each platform and version - it does a hunt from current position thing, to enhance the natural locality, and therefore increase cache hits and improve performance). As such it is not something that could or ever will be changed, but something for you to not do. Rather, if at all possible, stick with one name via IupSetHandle, and obtain any other information you might need via a standard (custom) attribute, such as "PROPERNAME", "MAIDENNAME", "NICKNAME" etc. In other words any string that IUP will otherwise quietly ignore, unlike say "SIZE" or "FONT", which it actually tries to do something with.
