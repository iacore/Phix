gFill

Definition: include xpGUI.e

gdx id = gFill(string attributes="", dword_seq args={})
Description: Creates void element, which dynamically occupies empty spaces always trying to expand itself. Its parent should be an gHbox or an gVbox, or else this type of expansion will not work. If an EXPAND is set on at least one of the other children of the box, then the fill expansion is ignored.

For more information on the attributes and args parameters see gSetAttributes.

It does not have a native representation.

Returns: the identifier of the created element.
pwa/p2js: Supported, however may yet be deprecated in favour of a justify-content:space-between compatible approach.
See Also: gHbox, gVbox
Attributes:
? EXPAND (non inheritable) (read-only) If User size is not defined, then when inside a gHbox EXPAND is HORIZONTAL, when inside a gVbox EXPAND is VERTICAL. If User size is defined then EXPAND is NO.
? SIZE (non inheritable) Defines the width, if inside a gHbox, or the height, if it is inside a gVbox. The standard format "wxh" can also be used, but width will be ignored if inside a gVbox and height will be ignored if inside a gHbox (since 3.3). When consulted behaves as the standard SIZE attribute.
