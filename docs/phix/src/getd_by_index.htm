getd_by_index

Definition: object data = getd_by_index(integer node, integer tid=1)
Description: Retrieve the data corresponding to the specified node index, a result from getd_index().
Comments: The getd() function returns 0 for non-existent keys: if that might be a valid value you must test whether the result of getd_index() is zero, and if not that result can be used to (quickly, without another search) retrieve the associated data using getd_by_index().

Obviously, you should not hang on to the results from getd_index() for very long; after several deld()/setd() that node index could easily end up on a free list or get recycled. In some cases a lock may be required over a getd_index()/getd_by_index() pair to prevent interference by another thread.

Note that getd_by_index(0) currently always simply returns 0, but may be changed to trigger a fatal error in some future release.
Example: 
?getd_by_index(getd_index("key"))
See Also: getd, getd_index
