?

Syntax: ? object x
Description: This is just a shorthand way of saying: print(1, x) - i.e. printing the value of an expression to the standard output.
Example: 
? sq_add({1, 2},{3, 4})  -- displays {4, 6}
See Also: print
Technicalia ? differs slightly from print() as it adds new-lines to make the output more readable on your screen, or wherever you have directed standard output.

Currently (see builtins\VM\psprintN.e\allascii) print(1,x) displays double quotes around strings. If the compiler knows that x is a string (see pmain.e/DoQu), it invokes puts() directly and no double quotes are displayed. As ? is normally used for quick and dirty diagnostics only, it is assumed that this minor discrepancy is of no importance, however should it matter to you then perhaps you can change DoQu or allascii. A longer term fix would require adding an option setting to p.ini or somewhere similar, and perhaps ship psprintN.e/psprintNq.e and select which one to actually include based on said flag, perhaps as per ptok.e/includeFile/"--pHeapD hack:". Which would all be an awful waste of effort should no-one really need it. If you want consistency, then quite simply and obviously you should just always use print(1,x).
