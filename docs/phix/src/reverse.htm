reverse

Definition: sequence res = reverse(sequence src, sequence pFromTo={1,-1})
Description: Reverse the order of elements in a sequence.
Comments: A new sequence is created where the top-level elements appear in reverse order compared to the original sequence.

Optionally the start and end points can be specified, with negative values counting left from the end of the input sequence. See Technicalia for compatibility issues with OpenEuphoria.
Example 1: 
reverse({1,3,5,7})              -- {7,5,3,1}
Example 2: 
reverse({{1,2,3}, {4,5,6}})     -- {{4,5,6}, {1,2,3}}
Example 3: 
reverse({99})                   -- {99}
Example 4: 
reverse("reverse.exw",{3,-6})   -- "resreve.exw"
reverse("reverse.exw",{3,+6})   -- ditto
-- (it is coincidental that -6 and +6 map to the same point in an 11 char string)
Example 5: 
reverse("123.456",{-3,-1})      -- "123.654"
--reverse("123.456",5,0)        -- OpenEuphoria equivalent/compilation error
Example 6: 
reverse({})                     -- {}
See Also: append, prepend, repeat
Technicalia The OpenEuphoria version of reverse uses a 0-based version of negative indexes whereby 0 denotes the last element, -1 the second to last element, and so on. Also, it does not support negative start positions. I briefly considered making the Phix version compatible with that nonsense, then decided against it, and made start/end a sequence (in OpenEuphoria they are two separate integers) to highlight any incompatibilities. See example 5. Only reverse(s), with no start/end, is compatible; if you need the latter I would suggest writing/using an application-specific reverse_subset(sequence src, integer pFrom, integer pTo) routine, based on the one in builtins\misc.e.
