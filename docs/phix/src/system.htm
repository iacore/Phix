system

Syntax: system(string st, integer mode=2)
Description: Pass a command string st to the operating system command interpreter.
Comments: Uses %COMSPEC% /C, which means it has the same features (redirection, wildcards, etc) and is subject to the same limitations (eg it was 126 chars on Win98, but has grown to 8000 chars on Win7) as manually using an MS-DOS console window.

NOTE: Specifying a windows GUI app appears to hang the console, no idea why. If that happens try using sysproc.

The mode parameter is a bit of a legacy feature and not guaranteed to work exactly as per this documentation. The default value of 2 ("do nothing") should suffice. It indicates the manner in which to return from the call to system():

When mode is 0, the previous graphics mode is restored and the screen is cleared.

When mode is 1, a beep sound will be made and the program will wait for the user to press a key before the previous graphics mode is restored.

When mode is 2 (the default), the graphics mode is not restored and the screen is not cleared. This should only be used when it is known that the command executed by system() will not change the graphics mode.

You can use Euphoria as a sophisticated "batch" (.bat) language by making calls to system() and system_exec().

system() will start a new DOS or Linux shell.

system() allows you to use command-line redirection of standard input and output in the command string st.

If you cannot decide between system, sysproc, system_exec, and sysexec, this table may help.
Example 1: 
system("copy temp.txt a:\\temp.bak", 2)
-- note use of double backslash in literal string to get a single backslash
Example 2: 
system("p.exe \\test\\myprog.ex < indata > outdata", 2)
-- executes myprog by redirecting standard input and output
See Also: sysproc, system_open, system_exec, dir, current_dir, getenv, command_line
